## Keyword Implementation Checklist

### Before Deployment

- [ ] Review all keywords for relevance
- [ ] Check keyword density (1-3% for primary keywords)
- [ ] Verify no keyword stuffing
- [ ] Ensure natural language flow
- [ ] Test title/description length
  - Title: 50-60 characters
  - Description: 150-160 characters
- [ ] Validate Open Graph tags
- [ ] Check schema markup with tool

- [ ] Update meta keywords tags

## Keyword Update Process

### When to Update Keywords

1. **Quarterly Review**
   - Check performance data
   - Identify new opportunities
   - Remove underperforming keywords

2. **After Content Updates**
   - Add keywords for new content
   - Update if page focus changes
   - Adjust for seasonal trends

3. **Competitive Analysis**
   - Monitor competitor keywords
   - Identify gaps
   - Find new ranking opportunities

### How to Update Keywords

1. **Edit SEO-KEYWORDS.md**
   - Update keyword lists
   - Change priorities
   - Add new keywords
   - Track in Current Status column

2. **Update HTML Files**
   - Edit meta keywords tag
   - Update title tag
   - Update description
   - Update Open Graph tags
   - Ensure proper keyword density (0.5-3%)

3. **Validate Before Deploy**
   ```bash
   npm run build:seo
   ```
   - Review SEO validation report
   - Fix warnings/errors reported
   - Check dist/seo-build-report.json

4. **Rebuild & Deploy**
   ```bash
   npm run deploy:seo
   ```

5. **Monitor Results**
   - Google Search Console
   - seo-build-report.json in dist
   - PageSpeed Insights
   - Analytics

## SEO Validation Workflow

### Quick Build with SEO Check

```bash
npm run build:seo
```

**Output includes:**
- Minified HTML/CSS/JS
- SEO validation for each page
- Keyword density analysis
- Meta tag validation
- `seo-build-report.json` with full report

### Deploy with SEO Validation

```bash
npm run deploy:seo
```

**Process:**
1. Convert images to WebP
2. Run SEO-enhanced build
3. Validate all keyword requirements
4. Git commit with SEO message
5. Push to GitHub Pages

### SEO Report File

After `npm run build:seo`, check: `dist/seo-build-report.json`

Contains:
```json
{
  "timestamp": "ISO-8601 timestamp",
  "totalSize": "bytes",
  "files": "count",
  "seoWarnings": [],
  "seoPassed": true/false
}
```

## Best Practices

### Do ✅

- Use keywords naturally in content
- Target 1-3 primary keywords per page
- Include related keywords
- Update keywords regularly
- Monitor search performance
- Track ranking positions
- Test with tools before deployment
- Document all changes

### Don't ❌

- Keyword stuff or overuse
- Use irrelevant keywords
- Copy competitor keywords
- Ignore user intent
- Neglect mobile keywords
- Forget to test changes
- Ignore ranking data
- Update without tracking
